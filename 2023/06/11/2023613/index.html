git工具上传使用

|阅读量:

本地文件夹使用git上传github

1全局部署GitHub用户名和邮箱

2对本地项目文件夹初始化

3添加项目文件

4查看本地提交状态

5.把文件提交到仓库

6.连接远程仓库

7.修改分支的名字

8.上传代码到GitHub仓库

9github不再支持密码验证上传

10生成自己的token，用来代替密码

登录github->setting->Developer setting->Generate new token

idea使用git上传github

1进入IDEA

Settings – Version Control – Git，选择Git的安装路径为bin目录下的git.exe可执行文件

2选择左侧Git下方的GitHub – Log In with Token

3将刚才生成的token粘贴进去以后Add accoun

4成功后会看到自己的account

5桌面右击，因为已经安装了Git，所以应该会有个Git Bash Here，点击它

6部署GitHub用户名和邮箱

7在用户文件夹下，应该会生成一个.gitconfig文件，用notepad++或者记事本打开查看是否有自己的account信息

8IDEA打开你的项目，Git – GitHub – Share Project on GitHub

9点击Share等待成功即

本地使用git上传gitee

同上传github基本相同

idea使用git上传gitee

1vcs->import into Version Control->Create Git Respository

2选择要上传的项目

3鼠标右键项目git->add添加或者右上角√add

4鼠标右键项目git->commit或者右上角√commit

5可以看到项目结构，然后添加解释信息

6出现警告，忽略继续commit

7到此，只是将项目提交到了本地仓库，还没有推到远端

8push操作，填写gitee仓库url

9检查是否成功
