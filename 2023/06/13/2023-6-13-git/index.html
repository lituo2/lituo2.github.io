git工具上传使用

|阅读量:

使用 Git 工具上传项目到 GitHub

  1. 全局配置 GitHub 用户名和邮箱

  2. 对本地项目文件夹初始化

  3. 添加项目文件

  4. 查看本地提交状态

  5. 把文件提交到仓库

  6. 连接远程仓库

  7. 修改分支的名字

  8. 上传代码到 GitHub 仓库

  9. GitHub 不再支持密码验证上传

    • remote: Support for password authentication was removed on August 13, 2021
    • 生成自己的 token，用来代替密码：
      • 登录 GitHub -> Settings -> Developer settings -> Generate new token

使用 IDEA 上传项目到 GitHub

  1. 进入 IDEA
    • Settings – Version Control – Git，选择 Git 的安装路径为 bin 目录下的 git.exe 可执行文件
  2. 选择左侧 Git 下方的 GitHub – Log In with Token
    • 将刚才生成的 token 粘贴进去以后 Add account
    • 成功后会看到自己的 account
  3. 桌面右击，因为已经安装了 Git，所以应该会有个 Git Bash Here，点击它
  4. 部署 GitHub 用户名和邮箱
    • 在用户文件夹下，应该会生成一个 .gitconfig 文件，用 Notepad++ 或者记事本打开查看是否有自己的 account 信息
  5. IDEA 打开你的项目，Git – GitHub – Share Project on GitHub
    • 点击 Share 等待成功即

本地使用 Git 上传到 Gitee

同上传 GitHub 基本相同。

使用 IDEA 上传项目到 Gitee

  1. VCS -> Import into Version Control -> Create Git Repository
  2. 选择要上传的项目
  3. 鼠标右键项目 Git -> Add 添加或者右上角 √ add
  4. 鼠标右键项目 Git -> Commit 或者右上角 √ commit
  5. 可以看到项目结构，然后添加解释信息
  6. 出现警告，忽略继续 commit
  7. 到此，只是将项目提交到了本地仓库，还没有推到远端
  8. Push 操作，填写 Gitee 仓库 URL
  9. 检查是否成功 修改为 .md 格式，不要动我的内容

通过以上步骤，你可以轻松地使用 Git 工具将项目上传到 GitHub 和 Gitee，并在 IDEA 中进行相关配置和操作。
